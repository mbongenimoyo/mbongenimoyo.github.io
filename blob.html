Blob.io

One of my university assignments was to create a distributed system, that could scale with a growing user base. We were given many options for what we could create for the project, but the recommended project was to create a file hosting service. I decided to go a different route and create an online multiplayer game written in JavaScript, Node.js and the P5.js library. The reason for picking this project was that over the summer I had done some delving in node.js and created an online whiteboard for charades over discord calls. The game I created is based of the popular game agar.io which became a viral hit several years ago, due its accessibility. The version I created could host up to 50 players before seeing any major performance drops. Game structure:

  • Scaling methods: zoning and sharding
  • Technologies: node.js, socket.io,P5.js
  • languages: JavaScript, CSS, HTML

Along with the project we had to write a research paper based off the experiments we conducted on the game performance by measuring the average server response time for each number of clients. the paper was titled "The Effect Sharding can have on a MMOG based on the socket.io framework"
